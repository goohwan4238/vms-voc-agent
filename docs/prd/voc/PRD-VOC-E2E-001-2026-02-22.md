# PRD: 사용자 프로필 이메일 표시 기능

**문서 버전**: 1.0.0
**작성일**: 2026-02-22
**상태**: 검토 대기

## 1. 개요

### 1.1 배경

VMSWorks의 사용자 프로필 페이지에는 현재 이름, 부서, 역할 등 기본 정보만 표시되며 이메일 주소가 누락되어 있다. `users` 테이블의 `email` 컬럼은 이미 존재하나 프로필 UI 및 API 응답에 포함되지 않아 사용자가 타 구성원의 연락처를 시스템 내에서 확인할 수 없는 불편이 발생하고 있다.

### 1.2 목적

- 프로필 조회 시 이메일 주소를 화면에 표시하여 사용자 간 커뮤니케이션 효율 개선
- 개인정보보호법(PIPA) 준수를 전제로 본인/타인 조회 시 노출 범위를 차등 적용
- 신규 DB 스키마 변경 없이 기존 데이터를 안전하게 활용

### 1.3 범위

| 포함 | 제외 |
|------|------|
| 프로필 조회 API 응답에 `email` 필드 추가 | 이메일 수정/변경 기능 |
| 본인 조회 시 전체 이메일 표시 | 이메일 인증 플로우 변경 |
| 타인 조회 시 마스킹 처리 | 관리자 이메일 일괄 조회 기능 |
| 비로그인 사용자 접근 차단 | 이메일 기반 알림 시스템 |

### 1.4 정책 결정 (착수 전제 조건)

> **⚠️ 기획/법무 확정 필요**

본 PRD는 아래 정책을 기준으로 작성되었다. 착수 전 확정이 필요하다.

| 조회 주체 | 표시 형식 | 예시 |
|-----------|-----------|------|

| 로그인한 타인 | 마스킹 처리 | `us**@example.com` |
| 비로그인 | 차단 (401) | — |

## 3. 사용자 스토리

### US-01: 본인이 자신의 프로필을 조회하는 경우

```
As a 로그인된 사용자
I want to 내 프로필 페이지에서 이메일 주소를 확인하고 싶다
So that 내 계정 정보를 한 곳에서 확인할 수 있다

Given: 사용자가 로그인 상태이다
When:  본인의 프로필 페이지(/profile/me 또는 /profile/{self_id})를 요청한다
Then:  이메일 주소가 마스킹 없이 전체 표시된다 (e.g. user@example.com)
```

### US-02: 로그인한 사용자가 타인의 프로필을 조회하는 경우

```
As a 로그인된 사용자
I want to 다른 구성원의 프로필 페이지에서 이메일을 확인하고 싶다
So that 업무 연락처를 시스템 내에서 찾을 수 있다

Given: 사용자가 로그인 상태이다
When:  타인의 프로필 페이지(/profile/{other_id})를 요청한다
Then:  이메일 주소가 마스킹 처리되어 표시된다 (e.g. us**@example.com)
And:   전체 이메일은 노출되지 않는다
```

### US-03: 비로그인 사용자가 프로필을 조회하는 경우

```
As a 비로그인 사용자
I want to 특정 사용자의 프로필 URL에 접근한다
So that (악의적 시도) 이메일 정보를 수집하려 한다

Given: 사용자가 비로그인 상태이다
When:  프로필 API 또는 페이지에 접근한다
Then:  401 Unauthorized 응답을 받는다
And:   이메일 데이터는 일절 노출되지 않는다
```

### US-04: 관리자가 사용자 목록을 조회하는 경우

```
As a 관리자 역할의 로그인된 사용자
I want to 사용자 관리 화면에서 이메일을 확인하고 싶다
So that 계정 관리 업무를 수행할 수 있다

Given: 관리자로 로그인 상태이다
When:  사용자 관리 목록 API를 요청한다
Then:  전체 이메일이 표시된다 (관리자는 마스킹 제외)
```


### 4.2 이메일 마스킹 유틸리티

```python
# app/utils/masking.py (신규)
def mask_email(email: str) -> str:
    """
    이메일 로컬 파트의 3번째 문자 이후를 ** 로 마스킹한다.
    e.g. user@example.com  → us**@example.com
         a@example.com     → a***@example.com
    """
    if not email or '@' not in email:
        return ''
    local, domain = email.split('@', 1)
    visible = local[:2] if len(local) >= 2 else local[:1]
    masked  = visible + '*' * max(2, len(local) - len(visible))
    return f"{masked}@{domain}"
```

## 5. API 명세

### 5.1 프로필 조회 API (수정)

| 항목 | 내용 |
|------|------|
| **Endpoint** | `GET /api/users/<int:user_id>/profile` |
| **인증** | `@login_required` (Flask-Login) |
| **권한** | 로그인 사용자 전체 접근 가능 |

**요청**

```http
GET /api/users/42/profile
Authorization: Session Cookie (Flask-Login)
```

**응답 — 본인 조회 (200 OK)**

```json
{
  "id": 42,
  "username": "john_doe",
  "email": "john.doe@company.com",
  "email_masked": false,
  "display_name": "John Doe",
  "department": "개발팀",
  "role": "member",
  "avatar_url": "/static/avatars/42.png",
  "created_at": "2024-01-15T09:00:00Z"
}
```

**응답 — 타인 조회 (200 OK)**

```json
{
  "id": 43,
  "username": "jane_doe",
  "email": "ja**@company.com",
  "email_masked": true,
  "display_name": "Jane Doe",
  "department": "기획팀",
  "role": "member",
  "avatar_url": "/static/avatars/43.png",
  "created_at": "2024-02-01T10:00:00Z"
}
```

**응답 — 비로그인 (401 Unauthorized)**

```json
{
  "error": "Authentication required",
  "code": 401
}
```

**상태 코드 정리**

| 상태 코드 | 조건 |
|-----------|------|
| 200 | 정상 조회 |
| 401 | 비로그인 접근 |
| 403 | 접근 권한 없음 (향후 확장 대비) |
| 404 | 존재하지 않는 user_id |

## 6. UI 화면 명세

### 6.1 기술 선택 근거

| 화면 | 기술 | 이유 |
|------|------|------|
| 프로필 상세 페이지 | **기존 기술 유지** | 기존 구현 방식에 따름 (Jinja2 or Vue.js) |
| 이메일 표시 컴포넌트 | 인라인 수정 | 신규 컴포넌트 불필요, 텍스트 필드 추가 수준 |

> 프로필 페이지가 Vue.js SPA라면 6.2-A를, Jinja2 템플릿이라면 6.2-B를 적용한다.

### 6.2-A Vue.js SPA 프로필 컴포넌트 (신규 화면인 경우)

```vue
<!-- app/static/vue/components/UserProfile.vue -->
<template>
  <div class="profile-card">
    <div class="profile-field">
      <label class="field-label">이메일</label>
      <div class="field-value">
        <span class="email-text">{{ profile.email }}</span>
        <span v-if="profile.email_masked" class="badge badge-masked">
          마스킹됨
        </span>
      </div>
    </div>
  </div>
</template>

<script>
export default {
  name: 'UserProfile',
  props: {
    userId: { type: Number, required: true }
  },
  data() {
    return { profile: {} }
  },
  async created() {
    const res = await fetch(`/api/users/${this.userId}/profile`, {
      credentials: 'include'
    })
    if (res.ok) {
      this.profile = await res.json()
    }
  }
}
</script>
```

### 8.2 Blueprint 라우트 구현 패턴

```python
# app/blueprints/users/routes.py

from flask import jsonify
from flask_login import login_required, current_user
from app.models.user import User
from app.utils.masking import mask_email
from . import users_bp  # Blueprint 인스턴스


@users_bp.route('/api/users/<int:user_id>/profile', methods=['GET'])
@login_required
def get_user_profile(user_id: int):
    user = User.query.get_or_404(user_id)

    is_self  = (current_user.id == user_id)
    is_admin = (current_user.role == 'admin')

    # 이메일 노출 정책 적용
    if is_self or is_admin:
        email        = user.email
        email_masked = False
    else:
        email        = mask_email(user.email)
        email_masked = True

    return jsonify({
        'id':           user.id,
        'username':     user.username,
        'email':        email,
        'email_masked': email_masked,
        'display_name': user.display_name,
        'department':   user.department,
        'role':         user.role,
        'avatar_url':   user.avatar_url,
        'created_at':   user.created_at.isoformat(),
    }), 200


@users_bp.route('/api/users/me/profile', methods=['GET'])
@login_required
def get_my_profile():
    """본인 프로필 — 항상 전체 이메일 표시"""
    return get_user_profile(current_user.id)
```

### 8.3 Marshmallow 스키마 패턴 (사용 중인 경우)

```python
# app/blueprints/users/schemas.py

from marshmallow import Schema, fields, post_dump
from app.utils.masking import mask_email


class UserProfileSchema(Schema):
    id           = fields.Int(dump_only=True)
    username     = fields.Str(dump_only=True)
    email        = fields.Str(dump_only=True)
    email_masked = fields.Bool(dump_only=True)
    display_name = fields.Str(dump_only=True)
    department   = fields.Str(dump_only=True)
    role         = fields.Str(dump_only=True)
    avatar_url   = fields.Str(dump_only=True)
    created_at   = fields.DateTime(dump_only=True)

    def dump_with_context(self, user, *, is_self: bool, is_admin: bool):
        data = self.dump(user)
        if not (is_self or is_admin):
            data['email']        = mask_email(data['email'])
            data['email_masked'] = True
        else:
            data['email_masked'] = False
        return data
```

### 9.2 API 통합 테스트

```python
# tests/api/test_user_profile.py
import pytest


class TestGetUserProfile:

    def test_본인_조회_이메일_전체_표시(self, client, logged_in_user):
        res = client.get(f'/api/users/{logged_in_user.id}/profile')
        assert res.status_code == 200
        data = res.get_json()
        assert data['email'] == logged_in_user.email
        assert data['email_masked'] is False

    def test_타인_조회_이메일_마스킹(self, client, logged_in_user, other_user):
        res = client.get(f'/api/users/{other_user.id}/profile')
        assert res.status_code == 200
        data = res.get_json()
        assert '**' in data['email']
        assert data['email_masked'] is True
        assert data['email'] != other_user.email  # 전체 이메일 미노출 확인

    def test_비로그인_접근_401(self, client, other_user):
        res = client.get(f'/api/users/{other_user.id}/profile')
        assert res.status_code == 401

    def test_관리자_타인_조회_전체_표시(self, client, admin_user, other_user):
        client.login(admin_user)
        res = client.get(f'/api/users/{other_user.id}/profile')
        assert res.status_code == 200
        data = res.get_json()
        assert data['email'] == other_user.email
        assert data['email_masked'] is False

    def test_존재하지_않는_사용자_404(self, client, logged_in_user):
        res = client.get('/api/users/99999/profile')
        assert res.status_code == 404

    def test_me_엔드포인트_전체_표시(self, client, logged_in_user):
```
