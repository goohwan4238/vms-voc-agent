# PRD: VOC 상세 페이지 깜빡임 개선

---

## 1. 개요

### 배경
VOC 목록 화면에서 상세 페이지로 전환할 때, 이전 VOC의 데이터 또는 빈 화면이 순간적으로 노출되는 **Flash of Incorrect Content(FOIC)** 현상이 발생하고 있다. 이는 Vue.js SPA 환경에서 상세 컴포넌트 마운트 시점과 API 응답 수신 시점 사이의 갭으로 인해 stale state가 렌더링되기 때문이다.

### 목적
VOC 상세 페이지 진입 시 잘못된 중간 상태 노출을 제거하고, 로딩 중 상태를 명확한 UI(스켈레톤 또는 인디케이터)로 대체하여 사용자 경험을 개선한다.

### 범위
- **포함**: VOC 상세 Vue 컴포넌트의 상태 초기화 및 로딩 처리 개선
- **포함**: 목록 → 상세 전환 시 이전 상태 초기화
- **제외**: VOC 데이터 모델 변경, DB 스키마 변경, 백엔드 API 신규 추가
- **제외**: 전역 Vuex/Pinia store 구조 전면 리팩토링

## 3. 사용자 스토리

### Story 1: 목록에서 상세로 전환 시 깜빡임 없음

```
Given: 사용자가 VOC 목록 페이지에 있고, 여러 VOC 항목이 표시되어 있다
When:  특정 VOC 행을 클릭하여 상세 페이지로 이동한다
Then:  이전 VOC 데이터 또는 빈 화면이 노출되지 않고,
       즉시 로딩 스켈레톤 UI가 표시된 후 정확한 데이터로 교체된다
```

### Story 2: 상세 페이지 간 빠른 전환

```
Given: 사용자가 VOC-001 상세 페이지를 보고 있다
When:  브라우저 뒤로가기 후 즉시 VOC-002 상세 페이지로 진입한다
Then:  VOC-001의 데이터가 VOC-002 로딩 전 순간이라도 노출되지 않는다
       스켈레톤 UI가 표시되고 VOC-002 데이터로 정확히 교체된다
```

### Story 3: 네트워크 지연 환경에서 로딩 상태 표시

```
Given: 사용자가 느린 네트워크 환경에서 VOC 상세 페이지에 진입한다
When:  API 응답이 1초 이상 지연된다
Then:  스켈레톤 UI 또는 로딩 인디케이터가 응답 완료 전까지 유지된다
       응답 완료 시 데이터가 자연스럽게 교체된다
```

### 8.2 핵심 수정 패턴 — `VocDetail.vue`

**현재 문제 패턴 (추정)**
```javascript
// ❌ 문제: id 변경 시 이전 voc 데이터가 초기화되지 않은 채 렌더링됨
export default {
  data() {
    return { voc: null }
  },
  async created() {
    await this.fetchVoc(this.$route.params.id)
  },
  methods: {
    async fetchVoc(id) {
      const res = await axios.get(`/api/voc/${id}`)
      this.voc = res.data
    }
  }
}
```

**수정 패턴**
```javascript
// ✅ 수정: watch를 활용해 id 변경 시 즉시 상태 초기화
export default {
  data() {
    return {
      voc: null,
      isLoading: false,
      error: null,
    }
  },

  async created() {
    await this.loadVoc(this.$route.params.id)
  },

  watch: {
    // 같은 컴포넌트 재사용 시 (라우터가 컴포넌트를 재생성하지 않는 경우)
    '$route.params.id'(newId) {
      this.loadVoc(newId)
    }
  },

  methods: {
    async loadVoc(id) {
      // 핵심: 이전 데이터 즉시 초기화 후 로딩 시작
      this.voc = null
      this.error = null
      this.isLoading = true

      try {
        const res = await axios.get(`/api/voc/${id}`)
        this.voc = res.data
      } catch (err) {
        this.error = err.response?.status === 404
          ? '존재하지 않는 VOC입니다.'
          : '데이터를 불러오는 중 오류가 발생했습니다.'
      } finally {
        this.isLoading = false
      }
    }
  }
}
```

**템플릿 수정 패턴**
```html
<template>
  <div class="voc-detail">
    <router-link :to="{ name: 'VocList' }">← 목록으로</router-link>

    <!-- 로딩 중: 스켈레톤 -->
    <div v-if="isLoading" class="voc-detail__skeleton">
      <div class="skeleton skeleton--title"></div>
      <div class="skeleton skeleton--meta"></div>
      <div class="skeleton skeleton--body"></div>
    </div>

    <!-- 에러 -->
    <div v-else-if="error" class="voc-detail__error">
      <p>⚠ {{ error }}</p>
      <button @click="loadVoc($route.params.id)">다시 시도</button>
    </div>

    <!-- 데이터 로드 완료 -->
    <div v-else-if="voc">
      <!-- 기존 상세 내용 -->
    </div>
  </div>
</template>
```

## 9. 테스트 계획

### 9.1 백엔드 API 테스트 (pytest)

> API 변경 없음. 기존 상세 조회 API 정상 동작 회귀 검증만 수행.

```python
# tests/test_voc_api.py

def test_voc_detail_returns_correct_data(client, auth_headers, sample_voc):
    """정상 조회 시 200과 올바른 데이터 반환"""
    res = client.get(f'/api/voc/{sample_voc.id}', headers=auth_headers)
    assert res.status_code == 200
    data = res.get_json()
    assert data['id'] == sample_voc.id
    assert data['title'] == sample_voc.title

def test_voc_detail_not_found(client, auth_headers):
    """존재하지 않는 ID 요청 시 404 반환"""
    res = client.get('/api/voc/999999', headers=auth_headers)
    assert res.status_code == 404

def test_voc_detail_requires_auth(client):
    """미인증 요청 시 401 반환"""
    res = client.get('/api/voc/1')
    assert res.status_code in (401, 302)  # redirect to login
```

### 9.2 UI 테스트 시나리오 (Playwright)

```python
# tests/e2e/test_voc_detail_flash.py

def test_no_stale_data_on_navigation(page):
    """VOC-001 → 목록 → VOC-002 전환 시 VOC-001 데이터 노출 없음"""
    page.goto('/voc/1')
    page.wait_for_selector('[data-testid="voc-title"]')
    title_voc1 = page.text_content('[data-testid="voc-title"]')

    page.goto('/voc/list')
    page.click('[data-testid="voc-row-2"]')  # VOC-002 클릭

    # 로딩 직후 VOC-001 제목이 노출되면 안 됨
    # 스켈레톤이 먼저 보여야 함
    assert page.is_visible('[data-testid="voc-skeleton"]') or \
           page.text_content('[data-testid="voc-title"]') != title_voc1

    page.wait_for_selector('[data-testid="voc-title"]')
    assert page.text_content('[data-testid="voc-title"]') != title_voc1

def test_skeleton_visible_during_loading(page):
    """느린 네트워크에서 스켈레톤 UI 노출 확인"""
    page.route('**/api/voc/**', lambda route: (
        page.wait_for_timeout(800), route.continue_()
    ))
    page.goto('/voc/1')
    assert page.is_visible('[data-testid="voc-skeleton"]')
    page.wait_for_selector('[data-testid="voc-title"]')
    assert not page.is_visible('[data-testid="voc-skeleton"]')

def test_error_state_on_invalid_id(page):
    """존재하지 않는 ID 접근 시 에러 UI 노출"""
    page.goto('/voc/999999')
    page.wait_for_selector('[data-testid="voc-error"]')
    assert page.is_visible('[data-testid="voc-error"]')
```

### 9.3 엣지 케이스

| 케이스 | 예상 동작 |
|--------|-----------|
| 동일한 VOC를 다시 클릭 | 이미 표시 중인 데이터 유지, 불필요한 fetch 스킵 가능 |
| fetch 중 다른 VOC로 이동 | 이전 fetch 응답 무시 (race condition 방지, AbortController 또는 버전 토큰 활용) |
| 404 응답 후 다시 시도 버튼 클릭 | 동일 URL로 재요청 수행 |
| 로그인 세션 만료 후 상세 진입 | 401 응답 → 로그인 페이지 리다이렉트 (기존 인터셉터 동작 확인) |
| store에 잔존 데이터 있는 상태에서 라우터 진입 | `clearSelectedVoc` 호출로 즉시 초기화 |
