# PRD: 결재 문서 목록 검색 기능 추가

---

## 1. 개요

**문서 번호**: PRD-XXX  
**작성일**: 2026-02-22  
**작성자**: (담당자 입력)  
**상태**: 초안  
**우선순위**: Medium  

결재 문서 목록 페이지에서 사용자가 원하는 문서를 빠르게 찾을 수 없다는 VOC가 접수되었다. 현재는 전체 목록을 스크롤하거나 수동으로 탐색해야 하므로, 문서 수가 누적될수록 업무 효율이 저하된다. 본 PRD는 텍스트 검색, 날짜 범위 필터, 결재 상태 필터를 결재 문서 목록에 추가하여 이 문제를 해결하는 것을 목적으로 한다.

## 2. 목표

### 비즈니스 목표
- 결재 문서 탐색에 소요되는 시간 단축
- 결재 업무 처리 효율 향상

### 성공 지표
| 지표 | 기준 |
|------|------|
| 검색 결과 응답 시간 | 95th percentile 기준 2초 이내 |
| 검색 기능 사용률 | 출시 1개월 후 DAU 대비 30% 이상 |
| 관련 사용자 문의 감소 | 문서 탐색 관련 문의 50% 감소 목표 |

### 범위 외 항목 (Out of Scope)
- 첨부파일 내용 검색 (Full-Text 본문 검색)
- 저장된 검색 조건 즐겨찾기
- 검색 결과 엑셀 내보내기

## 3. 사용자 스토리

### US-001. 키워드 검색
> **일반 사용자**로서, 결재 문서 목록에서 **제목 또는 작성자 이름으로 키워드 검색**을 하고 싶다.  
> 그래야 **원하는 문서를 빠르게 찾아 업무를 처리**할 수 있다.

**수용 기준**
- 검색어 입력 후 Enter 또는 검색 버튼 클릭 시 결과가 반영된다.
- 제목과 작성자 필드를 동시에 검색한다.
- 검색어가 없을 경우 전체 목록을 반환한다.
- 검색 결과가 없을 경우 "검색 결과가 없습니다" 메시지를 표시한다.

---

### US-002. 날짜 범위 필터
> **일반 사용자**로서, **시작일과 종료일을 지정하여 특정 기간의 결재 문서만** 조회하고 싶다.  
> 그래야 **월별·분기별 업무 현황을 확인**할 수 있다.

**수용 기준**
- 시작일과 종료일을 각각 독립적으로 설정할 수 있다.
- 시작일이 종료일보다 늦을 경우 오류 메시지를 표시하고 조회하지 않는다.
- 날짜 미입력 시 전체 기간을 조회한다.

---

### US-003. 결재 상태 필터
> **일반 사용자**로서, **진행중 / 완료 / 반려 상태별로** 결재 문서를 필터링하고 싶다.  
> 그래야 **처리해야 할 문서와 완료된 문서를 구분하여** 관리할 수 있다.

**수용 기준**
- 상태 필터는 다중 선택이 가능하다.
- 필터 미선택 시 전체 상태를 조회한다.
- 선택된 필터 상태가 시각적으로 구분된다.

---

### US-004. 복합 조건 검색
> **일반 사용자**로서, **키워드·날짜·상태 조건을 조합**하여 검색하고 싶다.  
> 그래야 **정확도 높은 결과를 한 번에** 얻을 수 있다.

**수용 기준**
- 모든 필터 조건은 AND 조건으로 결합된다.
- 조건 변경 시 페이지가 자동으로 1페이지로 초기화된다.
- 현재 적용된 조건이 UI에 명확히 표시된다.

---

### US-005. 검색 조건 초기화
> **일반 사용자**로서, **적용된 모든 검색 조건을 한 번에 초기화**하고 싶다.  
> 그래야 **전체 목록으로 빠르게 돌아올** 수 있다.

**수용 기준**
- "초기화" 버튼 클릭 시 모든 필터와 검색어가 제거된다.
- 초기화 후 전체 목록이 즉시 표시된다.

## 4. 기능 명세

### 4.1 검색 UI 컴포넌트

| 컴포넌트 | 설명 | 비고 |
|----------|------|------|
| 검색 입력창 | 키워드 입력 (placeholder: "제목 또는 작성자 검색") | 최소 1자, 최대 100자 |
| 날짜 범위 선택기 | 시작일 / 종료일 Date Picker | 오늘 날짜 이후 선택 불가 |
| 상태 필터 | 체크박스 그룹 (진행중 / 완료 / 반려) | 다중 선택 허용 |
| 검색 버튼 | 조건 적용 및 조회 실행 | — |
| 초기화 버튼 | 모든 조건 초기화 | — |

### 4.2 검색 동작 명세

**키워드 검색**
- 검색 대상 필드: `title`(제목), `author_name`(작성자명)
- 검색 방식: 부분 일치 (포함 검색)
- 대소문자 구분 없음
- 작성자 검색 기준: 사용자 **이름** 기준 (동명이인 처리 정책은 사전 협의 필요 — 리스크 참조)

**날짜 필터**
- 적용 기준: 문서 `created_at` (생성일) 기준
- 시작일 00:00:00 ~ 종료일 23:59:59 범위로 조회

**상태 필터**
- `IN` 조건으로 처리 (다중 선택 시)
- 상태값 매핑:

| 화면 표시 | 내부 상태값 |
|-----------|------------|
| 진행중 | `IN_PROGRESS` |
| 완료 | `COMPLETED` |
| 반려 | `REJECTED` |

### 4.3 페이지네이션 연동

- 검색 조건 변경 시 항상 **1페이지로 초기화**
- URL 쿼리스트링에 검색 조건 반영 (브라우저 뒤로가기 시 조건 유지)
- 페이지당 노출 건수: 기존 정책 유지

## 6. 기술적 접근 방안

### 6.1 백엔드 API 변경

```
GET /approval/documents
  ?keyword={string}     // 제목·작성자 검색어 (optional)
  &from={YYYY-MM-DD}    // 시작일 (optional)
  &to={YYYY-MM-DD}      // 종료일 (optional)
  &status={string[]}    // 상태 (optional, 다중: status=IN_PROGRESS&status=COMPLETED)
  &page={number}        // 페이지 번호
  &size={number}        // 페이지 크기
```

### 6.2 DB 쿼리 설계

- **기본 전략**: 복합 인덱스 추가로 `LIKE` 풀스캔 방지
  - 권장 인덱스: `(status, created_at)`, `(title)`, `(author_id)`
- **성능 임계치 이상 시**: Full-Text Search(FTS) 전환 고려
  - MySQL: `FULLTEXT INDEX` + `MATCH ... AGAINST`
  - PostgreSQL: `tsvector` + `GIN` 인덱스
- **작성자 검색**: `users` 테이블 JOIN 후 `author_name LIKE ?` 처리

### 6.3 프론트엔드 구현

- 검색 조건을 URL 쿼리스트링으로 동기화하여 공유·북마크 가능하도록 처리
- 검색 입력창 디바운스(300ms) 적용 — 불필요한 API 호출 최소화
- 조건 변경 시 `page=1` 강제 리셋 처리

### 6.4 권한 처리 (사전 협의 필요)

> ⚠️ **개발 착수 전 확인 필요**

| 사용자 유형 | 검색 범위 | 협의 결과 |
|------------|----------|----------|
| 일반 사용자 | 본인 관련 문서만 조회 가능 여부 | 미결 |
| 관리자 | 전체 문서 조회 가능 여부 | 미결 |

## 7. 테스트 계획

### 7.1 기능 테스트

| 테스트 케이스 | 입력 | 기대 결과 |
|--------------|------|----------|
| TC-01 제목 키워드 검색 | keyword="출장" | 제목에 "출장" 포함된 문서만 반환 |
| TC-02 작성자 검색 | keyword="홍길동" | 작성자가 "홍길동"인 문서만 반환 |
| TC-03 날짜 범위 정상 | from=2026-01-01, to=2026-01-31 | 해당 기간 문서만 반환 |
| TC-04 날짜 범위 역순 | from=2026-02-01, to=2026-01-01 | 오류 메시지 표시, 조회 안 함 |
| TC-05 상태 단일 필터 | status=COMPLETED | 완료 상태 문서만 반환 |
| TC-06 상태 다중 필터 | status=COMPLETED&status=REJECTED | 완료+반려 문서 반환 |
| TC-07 복합 조건 | keyword="계획" + status=IN_PROGRESS | AND 조건으로 조합된 결과 반환 |
| TC-08 검색 결과 없음 | keyword="zzzzzzz" | "검색 결과가 없습니다" 메시지 표시 |
| TC-09 조건 초기화 | 초기화 버튼 클릭 | 전체 목록 반환, 모든 필터 해제 |
| TC-10 페이지 초기화 | 2페이지에서 조건 변경 | 1페이지로 자동 이동 |

### 7.2 성능 테스트

- 문서 10만 건 이상 환경에서 검색 응답 시간 2초 이내 확인
- 동시 접속 50명 기준 응답 성능 측정

### 7.3 권한 테스트

- 일반 사용자가 타인 문서를 검색으로 접근할 수 없음을 확인
- 관리자 권한 범위 검증 (협의 결과 확정 후 케이스 추가)
