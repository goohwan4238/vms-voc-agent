# PRD: 자재 입출고 이력 조회 화면 추가

---

## 1. 개요

### 배경
현재 자재 관리 모듈에서는 입출고 트랜잭션에 대한 이력 조회 전용 화면이 존재하지 않아, 담당자가 특정 기간·자재·유형별 이력을 확인하려면 여러 메뉴를 경유하거나 별도 요청이 필요한 상황입니다.

### 목적
날짜 범위, 자재코드, 입출고 유형을 복합 조건으로 검색할 수 있는 **자재 입출고 이력 조회 전용 화면**을 신규 메뉴로 추가하여, 자재 관리 운영 효율을 높입니다.

### 범위
- 신규 메뉴 및 조회 화면 개발
- 기존 입출고 트랜잭션 테이블 기반 Read-only 조회
- 기존 기능 변경 최소화

## 2. 목표

| # | 목표 | 측정 기준 |
|---|------|-----------|
| 1 | 입출고 이력 단일 화면 조회 제공 | 전용 메뉴 1개 신규 생성 |
| 2 | 복합 필터 검색 지원 | 날짜·자재코드·유형 3가지 조건 동시 적용 가능 |
| 3 | 대용량 이력 데이터 안정적 처리 | 페이징 처리, 최대 5만 건 조회 시 응답 5초 이내 |
| 4 | 기존 기능 영향 없음 | 회귀 테스트 통과율 100% |

## 3. 사용자 스토리

### US-01. 기간별 이력 조회
> **자재 담당자**로서, **특정 기간의 입출고 이력 전체를 조회**하고 싶다.  
> 왜냐하면 **월말 마감 및 감사 대응 시 기간별 트랜잭션 내역을 빠르게 확인**해야 하기 때문이다.

**인수 조건**
- 시작일~종료일 입력 후 조회 버튼 클릭 시 해당 기간 이력이 그리드에 표시된다.
- 날짜 범위 미입력 시 기본값으로 최근 1개월이 설정된다.

---

### US-02. 자재코드 단건 조회
> **자재 담당자**로서, **특정 자재코드의 전체 입출고 이력을 조회**하고 싶다.  
> 왜냐하면 **특정 자재의 재고 변동 원인을 추적**해야 하기 때문이다.

**인수 조건**
- 자재코드 입력 후 조회 시 해당 자재의 이력만 필터링되어 표시된다.
- 자재코드는 직접 입력 또는 자재 검색 팝업을 통해 선택 가능하다.

---

### US-03. 입출고 유형별 조회
> **재고 관리자**로서, **입고/출고/반품 등 유형별로 이력을 분리하여 조회**하고 싶다.  
> 왜냐하면 **유형별 거래량 파악 및 이상 거래 탐지**가 필요하기 때문이다.

**인수 조건**
- 유형 선택 드롭다운에서 단일 또는 복수 유형 선택이 가능하다.
- 전체 선택 옵션이 기본값으로 제공된다.

---

### US-04. 조회 결과 엑셀 다운로드
> **자재 담당자**로서, **조회된 이력 데이터를 엑셀 파일로 내려받고**싶다.  
> 왜냐하면 **보고서 작성 및 타 시스템 연계 작업**에 활용해야 하기 때문이다.

**인수 조건**
- 다운로드 버튼 클릭 시 현재 필터 조건의 전체 결과가 `.xlsx` 형식으로 내려받아진다.
- 최대 다운로드 건수는 10만 건으로 제한하며, 초과 시 안내 메시지를 표시한다.

## 5. 타 기능 영향도

### 5.1 연관 모듈

| 모듈 | 관계 | 영향 수준 |
|------|------|-----------|
| 자재 관리 (재고) | 메뉴 추가 | 낮음 — 메뉴 트리에 신규 항목 추가만 필요 |
| 입고 처리 | 데이터 소스 | 없음 — Read-only 조회만 수행 |
| 출고 처리 | 데이터 소스 | 없음 — Read-only 조회만 수행 |
| 재고 현황 화면 | 데이터 공유 | 없음 — 공통 테이블 조회이나 독립적 쿼리 |
| 권한 관리 | 메뉴 권한 등록 | 낮음 — 신규 메뉴 ID 권한 테이블 추가 필요 |

### 5.2 예상 연관 테이블

| 테이블 | 용도 | 변경 여부 |
|--------|------|-----------|
| `material_transaction` | 입출고 이력 메인 데이터 | 변경 없음 (조회만) |
| `material_master` | 자재코드·자재명 조인 | 변경 없음 |
| `warehouse_stock` | 창고 정보 조인 | 변경 없음 |
| `code_master` | 유형 코드명 변환 | 변경 없음 |
| `menu_permission` | 신규 메뉴 권한 등록 | **추가 필요** |

> 입출고 이력이 복수 테이블에 분산된 경우, 분석 단계에서 UNION 또는 VIEW 생성 여부를 결정합니다.

## 6. 기술적 접근 방안

### 6.1 쿼리 설계

```sql
-- 기본 조회 쿼리 구조 (예시)
SELECT
    mt.transaction_dt,
    mt.material_cd,
    mm.material_nm,
    mt.transaction_type,
    cm.code_nm    AS type_nm,
    mt.quantity,
    mt.unit_cd,
    wh.warehouse_nm,
    mt.created_by,
    mt.remark
FROM material_transaction mt
JOIN material_master    mm ON mt.material_cd    = mm.material_cd
JOIN warehouse_stock   wh ON mt.warehouse_cd   = wh.warehouse_cd
JOIN code_master       cm ON mt.transaction_type = cm.code_id
                         AND cm.code_group = 'TRANSACTION_TYPE'
WHERE mt.transaction_dt BETWEEN :start_dt AND :end_dt
  AND (:material_cd    IS NULL OR mt.material_cd     = :material_cd)
  AND (:type_list      IS NULL OR mt.transaction_type IN (:type_list))
ORDER BY mt.transaction_dt DESC
```

### 6.2 성능 최적화

- `material_transaction` 테이블의 `transaction_dt` 컬럼 인덱스 존재 여부 확인 → 없을 경우 인덱스 추가 검토
- 복합 인덱스 후보: `(transaction_dt, material_cd, transaction_type)`
- 페이징: `LIMIT / OFFSET` 또는 커서 기반 페이징 적용
- 엑셀 대용량 다운로드: 스트리밍 방식으로 메모리 부하 방지

### 6.3 화면 구현

- 기존 프로젝트 UI 프레임워크 및 그리드 컴포넌트 재사용
- 자재코드 검색 팝업은 기존 공통 팝업 재사용 (신규 개발 불필요)
- 날짜 범위 피커는 기존 공통 컴포넌트 재사용

### 6.4 권한 처리

- 신규 메뉴 ID를 권한 관리 테이블에 등록
- 기존 자재 관리 권한 그룹에 자동 포함되도록 설정 (정책 협의 필요)
- 이력 전체 조회 허용 범위는 역할(Role)별 정책 확인 후 적용

## 7. 테스트 계획

### 7.1 단위 테스트

| 테스트 항목 | 검증 내용 |
|-------------|-----------|
| 날짜 필터 | 시작일 > 종료일 입력 시 에러 메시지 표시 |
| 날짜 범위 제한 | 1년 초과 범위 선택 시 안내 메시지 표시 |
| 자재코드 검증 | 존재하지 않는 자재코드 입력 시 경고 표시 |
| 유형 필터 | 전체/단일/복수 선택 결과 정확성 |
| 빈 결과 처리 | 검색 결과 없을 때 "데이터가 없습니다" 안내 |

### 7.2 통합 테스트

| 테스트 항목 | 검증 내용 |
|-------------|-----------|
| 복합 필터 | 날짜 + 자재코드 + 유형 3가지 동시 적용 정확성 |
| 페이징 | 다음 페이지 이동 시 데이터 연속성 및 중복 없음 |
| 정렬 | 컬럼 헤더 클릭 시 오름/내림차순 전환 정확성 |
| 엑셀 다운로드 | 필터 조건 일치 데이터가 파일에 정확히 포함되는지 확인 |

### 7.3 성능 테스트

| 시나리오 | 기준 | 목표 |
|----------|------|------|
| 5만 건 조회 | 응답 시간 | 5초 이내 |
| 20명 동시 조회 | 서버 응답 | 정상 처리 |
| 10만 건 엑셀 다운로드 | 완료 시간 | 30초 이내 |

### 7.4 회귀 테스트

- 입고 처리, 출고 처리, 재고 현황 화면 기존 기능 정상 동작 확인
- 기존 메뉴 권한 구조 변경 없음 확인
